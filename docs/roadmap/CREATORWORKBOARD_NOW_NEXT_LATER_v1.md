# CreatorWorkboard — NOW / NEXT / LATER / NOT NOW v1

## Purpose

This document is the practical scope guard for the corrected CreatorWorkboard product.

It exists to prevent the connection-board direction from drifting back into:

- inbox thinking
- marketing-monitor thinking
- automation-first thinking
- full suite thinking

## Oordeel

The correct focus is:

- Route 1 first
- board before transport replacement
- human operators before AI assistance
- connection cards before chat transcripts
- control layer before suite expansion

## NOW

The following belongs in NOW.

### 1. Existing chat operations first

Build around the revenue machine that already exists.

That means:

- connect existing chat routes into the board
- reduce context loss in current work
- improve operator handoff
- improve routing clarity
- improve assignment clarity

### 2. Workspace and control layer hardening

Now means:

- handoff visibility
- handoff editing
- acknowledgement v1
- source context
- risk/policy visibility
- next action clarity
- quick actions into external systems

### 3. Minimal connection-card model

Now means:

- minimal work card
- creator linkage
- optional channel linkage
- source
- paywall state
- routing state
- assignee
- external thread reference
- next action
- light notes/handoff linkage

### 4. Queue basics

Now means:

- operator home / queue
- route and status filters
- blocked / stalled visibility
- missing handoff visibility
- missing acknowledgement visibility

### 5. Strict human-in-the-loop

Now means:

- human-operated service remains core
- AI may assist later, but does not replace operators

## NEXT

The following belongs in NEXT, only after NOW is working.

### 1. Connector refinement

- stronger external thread references
- more reliable route linking
- lightweight connection events
- better per-system quick actions

### 2. Better queue behaviour

- improved sorting and prioritisation
- cleaner blocked-state handling
- queue age and stall indicators
- better operational hygiene metrics

### 3. Narrow Route 2 expansion

- basic OnlyFans/Fansly context support
- outside-paywall to inside-paywall route transitions
- creator/operator collaboration states
- route-specific next actions

### 4. Cautious AI assistance

- summaries
- labels
- priority suggestions
- operator prep support

Only if:

- transparent
- reversible
- never autonomous

## LATER

The following belongs in LATER.

### 1. Deeper paywall route support

- richer OF/Fansly workflow support
- deeper state modeling
- more creator collaboration modes

### 2. Broader backoffice support

Only if it directly serves the board model and proven operations.

### 3. More connectors

Only after the first route is working well.

### 4. Richer history and activity layers

Only after the minimal work card and queue prove their value.

### 5. Possible conversation-layer ownership

Only if external transport remains the structural bottleneck after the board model is already proven.

## NOT NOW

The following is explicitly NOT NOW.

### Product traps

- new inbox as the core product
- replacing all external chats immediately
- AI-first positioning
- autonomous conversation engine
- DM bot framing
- analytics-first dashboard identity

### Suite traps

- full OF/Fansly suite
- broad backoffice
- full monetization engine
- multitenant SaaS expansion
- generic creator CRM expansion

### Architecture traps

- building transport before the board
- over-modeling before connection cards exist
- building for every future route at once
- connector overengineering before Route 1 proves useful

## Why this scope guard exists

CreatorWorkboard becomes weak if it tries to be all of these at once:

- chat product
- creator suite
- AI operator
- analytics monitor
- backoffice

CreatorWorkboard becomes strong if it first becomes one thing:

- the human-controlled board that makes conversation-driven conversion work manageable.

## Short decision rule

A proposed feature belongs in NOW only if it strongly improves at least one of these:

- context quality
- routing clarity
- handoff quality
- acknowledgement discipline
- operator flow
- Route 1 usability

If not, it probably does not belong in NOW.
