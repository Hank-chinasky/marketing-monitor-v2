## Verslag van wat we hebben gedaan

We hebben de deployment van `creatorworkboard-ops` op de VPS werkend gekregen en daarna de eerste gebruikers- en appdata ingericht.

Eerst is de applicatie opnieuw vanaf GitHub op de VPS geplaatst onder:

```text
/opt/commandcenter/apps/creatorworkboard-ops
```

Daarna hebben we de Docker deployment opgebouwd en gestart. De app startte correct op, migrations draaiden, static files werden verzameld en Gunicorn kwam online.

Vervolgens hebben we een belangrijk deploymentprobleem opgelost:
de container healthcheck faalde door een HTTP→HTTPS redirect op `/healthz/`, waardoor de container unhealthy bleef. We hebben de compose-healthcheck aangepast zodat deze de app correct controleert achter Traefik. Daarna werd de container healthy.

Daarna hebben we de Traefik-routering voor `ops.creatorworkboard.com` hersteld en bevestigd dat:

* de container op `cc_public` zit
* de Traefik-labels correct actief zijn
* de router lokaal en extern correct matcht

Vervolgens hebben we Traefik basic auth hersteld door een nieuw wachtwoord te zetten voor `traefik-auth@docker`. Daarmee werkt de eerste auth-laag weer correct.

Daarna hebben we het Django loginprobleem onderzocht. De Django user-authenticatie werkte al, maar browser-login faalde op CSRF. De oorzaak bleek niet Django zelf te zijn, maar de Traefik security header:

```text
Referrer-Policy: no-referrer
```

Die brak de CSRF-validatie op HTTPS-login. We hebben dit aangepast naar een bruikbare policy, waarna login via browser weer werkte.

Na herstel van de loginflow hebben we gebruikers ingericht:

* superuser `Superadmin`
* bestaande admin bevestigd
* operators als Django users aangemaakt

Daarna bleek dat creators niet direct aan Django `User` gekoppeld zijn, maar aan een apart `Operator` model. We hebben daarom:

* echte `Operator` records aangemaakt voor de operator-users
* een eerste `Creator` aangemaakt
* de creator gekoppeld aan `operator_manoel`

Daarmee is bevestigd dat:

* deployment werkt
* healthcheck werkt
* Traefik auth werkt
* Django login werkt
* CSRF werkt
* operator/business data werkt
* eerste creatorrecord correct is opgeslagen
